Write-Ahead Logging (WAL) : PostgreSQL écrit toutes les modifications dans le journal WAL avant de modifier les fichiers de données. Lors d'un COMMIT, le WAL est synchronisé sur disque (fsync). Les fichiers de données sont écrits plus tard lors du checkpoint. En cas de crash, les enregistrements WAL sont rejoués pour restaurer l'état cohérent.
